## 筹码峰（CYQ）作为第五维辅助指标（v2.1 新增 · 2026-06-16）

### 数据来源

通过 DFCF 妙想 API `data()` 查询 `筹码分布 获利盘比例`，返回四个关键指标：

| 指标 | 字段 key | 含义 |
|------|---------|------|
| 平均持仓成本 | `010000_CMPJCB` | 所有持股人的平均买入价（元） |
| 获利盘比例 | `010000_HLP` | 当前价之上有多少持仓在盈利（%） |
| 70%筹码集中度 | `010000_CMFB_461_JZD70` | 含70%筹码的价格区间宽度（%） |
| 90%筹码集中度 | `010000_CMFB_461_JZD90` | 含90%筹码的价格区间宽度（%） |

### 与四灯的配合

```
四灯全亮 + 当前价 < 均价 → 🟢 双重共振（最强信号）
四灯全亮 + 当前价 > 均价 → 🟡 四灯成立但获利盘有兑现压力
四灯不全 + 当前价 < 均价 → 🟡 筹码低位但时机未到
四灯不全 + 当前价 > 均价 → 🔴 都不满足
```

**筹码低位 = 安全边际好**：价格低于平均成本时，套牢盘不愿割肉→卖盘少→反弹阻力小。
**筹码高位 + 获利盘>70%**：获利盘累积→随时可能跑路→即使四灯亮也要警惕。

### 实战案例（澜起科技 2026.6）

```
均价 235 / 当前 228 / 获利 25.68% / 70%集中 17.72%
→ 价格在均价之下=筹码低位+安全边际好
→ 四灯全亮+BOLL收口+J=3.36 → 五维共振
```

### 用户看盘对照

- 东方财富 APP 默认显示 BOLL + 筹码峰（CYQ）
- 招商证券默认显示 BOLL-M（无筹码峰）

## 尾盘选股策略（华生·tail-market-scanner）

**核心理念：** 不讲题材板块，只看指标——14:50扫描，符合条件次日开盘卖。胜率略高于抛硬币，真正的优势在**赔率**（尾盘买次日卖，亏损小盈利可能大）。

**两步法：**
1. **海选层** — `clist/get` 全A股：涨幅2-9% + 换手>3% + 量比>1.5 + 去ST
2. **验证层** — `trends2/get` 逐只拉1分钟分时，三维评分（0-100）：
   - 均价线上方时间占比（40分）
   - 下午走势强度（30分）— 斜率+回撤惩罚
   - 尾盘资金托举（30分）— 14:30-14:50稳定性+成交量+支撑

**纪律（王富贵版）：**
- ⏰ 只在14:50跑，不在其他时间
- 🚫 不合格就空仓——不交易也是策略的一部分
- 📉 大盘共振下跌（三四千家跌）→ 空仓日
- 📈 科创板/创业板集体涨停（涨幅前30全>9%）→ 空仓日（温和上涨区间被消灭，不是策略失灵）
- ⚠️ 振幅>8%、疑似炸板 → 自动扣分

**空仓日识别（2026-06-04 实战验证）：** 当日科创板集体爆发，涨幅前30全>11%，20+只封20%涨停板。早上8只候选全部被市场极端化消灭——要么冲板（>9%淘汰）、要么炸板（如永冠新材涨停→收+0.7%）、要么被吸血回落。策略输出0候选是正确保护行为，非故障。

**数据源：** 东方财富公开API（`push2his.eastmoney.com`，HTTPS + urllib，无需密钥）。分钟线/分时数据是妙想Skills API不提供的。详见 → `references/eastmoney-api.md`

**⚠️ 网络坑：** WSL环境下 `requests` 库连 `push2his` 会 `RemoteDisconnected`，`urllib.request` 直连HTTPS可通。脚本已用 `urllib` 规避。`push2.eastmoney.com`（不带his）的HTTP/HTTPS均不可达，必须用 `push2his` 子域。

**脚本：** `scripts/tail_market_scanner.py`
```bash
python scripts/tail_market_scanner.py --top 20 --debug
```

**与主策略（BOLL+KDJ+MACD）的关系：** 尾盘策略是独立的短线工具，与中线回测策略互补——尾盘策略负责"今天买明天卖"的短线机会，中线策略负责"等信号→持仓→MACD死叉离场"的趋势交易。

## 投资框架

**基础设施型垄断**：单价极低但不可或缺，占成品成本<1%，缺了产线停摆。认证周期1-3年，客户不敢换→天然定价权。

**卖耗材 > 卖铲子**：设备卖一次吃十年，耗材用了就得一直买。国瓷卖钛酸钡给村田三星风华——不管谁赢都赚。

详见 → `references/investment-framework.md`

### 适用边界

- ✅ 几百亿~几千亿市值的高弹性标的
- ✅ BB中下轨+KDJ超卖+MACD力竭
- ✅ 有不可替代性的业绩股（认证+工艺+客户惯性）
- ❌ 万亿级大盘股（弹性不足）
- ❌ 纯概念/辟谣股（一字涨跌停跑不掉）
- ❌ 单边下跌趋势（等MACD拐头）

### 策略局限性（回测不代表未来）

回测基于历史数据，以下情况策略可能失灵：

| 风险 | 表现 | 实例 |
|---|---|---|
| **市场风格切换** | 长期单边下跌中，BOLL中轨一路下移，每次抄在中轨都被埋 | 2023-2024年A股阴跌期未覆盖 |
| **MACD延迟确认** | 死叉出现时已回吐大部分利润 | 蓝思科技第三笔 -30.4%（37→26） |
| **标的市值膨胀** | 同一只股票从几十亿涨到千亿，波动特征改变 | 佰维50→345，低位的信号质量≠高位的信号质量 |
| **行业不匹配** | 参数在存储/MLCC上优化，其他行业未必适用 | 卓胜微（射频）回测+14%，远低于存储标的 |

**核心原则：** 策略告诉你概率，不是保证。任何一笔交易都可能亏损——策略的价值在于长期胜率和盈亏比，不在于单笔对错。

### BOLL收口（Squeeze）作为四灯增强信号

当四灯买入信号出现时，检查BOLL带宽是否在收缩——上轨下弯+下轨上翘 = 波动率收敛 = 方向选择在即。

- **收口+四灯+价格贴下轨**：弹簧压紧，突破后往上概率大（下轨在托着价格，空头衰竭+多头蓄力）
- **张口+四灯+价格在下轨附近**：波动还在放大，方向不明，等收口再判断
- **收敛末端**：通常会有一根方向性K线，放量阳线是入场确认，跌破下轨是止损信号

实战示例（澜起科技 2026.6）：四灯全亮后BOLL从张口逐步收口（上轨↓下轨↑），配十字星+缩量——底部收敛形态，等放量阳线确认。

### 数据源故障时的实用策略

push2his 间歇性不可达 + dfcf data() 对A+H股返回混合价格（如澜起同时返回A股228和H股345两个价格序列）。当两个数据源都不可靠时：

1. **优先信任用户终端数据**——用户在看盘软件上的实时数据比API更准
2. **不要反复重试失败的数据源**——两次失败后直接用用户提供的数字计算
3. **web_search 可查收盘价**（雪球/新浪/Yahoo），但日级别K线细粒度数据仍以用户为准

## 异常与降级策略

数据来源优先级：**东方财富公开 API（`push2his`，quick_scan.py 首选）→ 妙想 API（`dfcf_finance.py data()`，降级方案）**。公开 API 返回完整 OHLC，可一次算完四灯；妙想 API 每条 query 只返回一个指标，需要三次独立调用。

### push2his 不可达时的妙想降级（重要）

当 quick_scan.py 报 `Remote end closed connection without response` 时，按以下步骤降级：

1. **单独查询收盘价**：`tool.data('688008 澜起 收盘价 2026-04-01至2026-06-10')`
2. **单独查询最高价**：同上但 query 为「最高价」
3. **单独查询最低价**：同上但 query 为「最低价」

每个查询返回的 table 结构中，实体 key 是**数字 ID**（如 `325898`），不是股票名称。headName 存日期（格式 `2026-06-09(日)`），值带「元」后缀。需要 `re.sub(r'[元港,日\(\)]', '', s)` 清洗后转 float。

**关键坑：** 一次 query 同时请求「收盘价 最高价 最低价」时，返回 2 个 table——table 0 是汇总统计（区间高低点 + 日期），table 1 只有收盘价序列。高点/低点不会出现在同一 query 中，必须分三次调用。妙想 data() **不支持 OHLC 四合一查询**。

数据反转：返回的序列是**最新在前**（降序），需要 `[::-1]` 反转为升序后才能计算指标。

### 妙想 data() 返回错误价格（A/H 股混淆）

当股票有 A+H 双重上市时（如澜起科技 688008 + 06809），`data()` 的 table 中可能同时包含 A 股和 H 股数据。H 股价格通常远高于 A 股（溢价），若误取 H 股价格，会导致 BOLL/MACD/KDJ 全部失真。

**防御措施：** 解析 table 时优先匹配含 `.SZ` /「A股」/股票代码（不含港）后缀的 key；若遇到 H 股价格（通常以「港」结尾），跳过。quick_scan.py 直接从 push2his 拉取，不存在此问题。**fallback_scan.py 已修复（2026-06-12）：** ① 跳过含「港元」的 table（妙想 data() 对 A+H 股票返回两个 table，Table 0=HKD、Table 1=CNY）；② `parse_prices()` 过滤含「港」的值；③ 补了缺失的 `import os`。

### 网络连通性

从 WSL/Linux 环境访问东方财富 API 时，`requests` 库可能连接失败（`RemoteDisconnected`），但 **`urllib.request.urlopen` 同 URL 可通**。所有调用东财公开接口的脚本（quick_scan、backtest、tail_market_scanner）必须使用 `urllib` 而非 `requests`。

**已验证可通的子域名：** `push2his.eastmoney.com`（HTTPS）。`push2.eastmoney.com`（HTTP）不可达。该域名**间歇性全站宕机**——服务端不可用，与客户端 SSL/网络/Tailscale 均无关。


**🔬 诊断流程（2026-06-12 验证）：**

```bash
# Step 1: 存活检测
curl -s --max-time 5 "https://push2his.eastmoney.com/api/qt/stock/kline/get?secid=1.688008&fields1=f1&fields2=f51&klt=101&fqt=0&end=$(date +%Y%m%d)&lmt=1"
```

**Step 2: 如果 curl 能通但 quick_scan.py 仍然报 error 102：** 运行 `scan()` 函数直接调用（绕过命令行入口的间歇问题）：
```python
python3 -c "exec(open('scripts/quick_scan.py').read().replace('if __name__', 'if False and __name__')); scan('1.688008')"
```

**Step 3: Python urllib 的 TLS 问题（WSL特有）：** `quick_scan.py` 已内置 `ssl._create_unverified_context()`（2026-06-12 修复）。症状：服务在线但 Python 报 `RemoteDisconnected`，curl 正常。根因：Python 默认 TLS 握手与东方财富 IIS 服务器不兼容。

若子技能或数据源不可用，按以下三段式 fallback：
1. `quick_scan.py`（push2his，含高/低/收价 → 完整 KDJ+BOLL+MACD）→ 首选
2. `quick_scan.py` 报 error 102 但 curl 能通 → exec() 直调 `scan()` 函数（绕过命令行入口间歇问题）
3. `fallback_scan.py`（妙想 API 收盘价 → numpy 算 BOLL/MACD，高/低价用 close±2% 估算 KDJ）→ push2his 不可达时
4. 裸调妙想 API 分别查询收盘价/最高价/最低价（三次 API 调用，完整 KDJ）→ fallback_scan 也失败时
4. 妙想 API 返回的时间序列数据提取模式（重要）：
   - `data()` 用中文"至"连接日期：`'688525 收盘价 2026-04-01至2026-06-10'`
   - 返回 `table` 字典，**entity key 是数字 ID（如 '325898'），不是股票代码或名称**
   - `headName` 存日期（格式 `'2026-06-09(日)'`），需 `replace('(日)','')` 清洗
   - 价格数组含 `'元'` 后缀，需 `replace('元','')` 后转 float
   - 日期顺序为最新在前，需 `[::-1]` 反转为时间顺序
   - 收盘价/最高价/最低价需分开三次查询，单次混合查询不返回多表注意：push2his 可能间歇性全站不可达（RemoteDisconnected），此时需走三级降级。

若子技能或数据源不可用，按以下三段式 fallback：

| 触发条件 | 一线修复 | 仍失败兜底 |
|---|---|---|
| dfcf API 无响应/返回空 | 重试1次，间隔3秒；切换备用数据源（web_search查行情） | 告知用户"数据不可用"，使用上次缓存数据（标注日期） |
| push2his RemoteDisconnected | ① curl 快速存活检测；② 在线 → `quick_scan.py` 内置 SSL fix 已处理；③ 宕机 → `fallback_scan.py <code> <name>`（妙想收盘价+估算KDJ） | 裸调妙想 API 分三次查收/高/低价完整算KDJ |
| quick_scan.py error 102（但 curl 能通） | exec() 直调 `scan()` 函数 | 同 push2his 降级 |
| 回测脚本运行失败 | 检查 Python 依赖（`pip list \| grep pandas`）；确认数据文件存在 | 降级为手工策略判断——人工看 KDJ/MACD/BOLL 日线图 |
| 回测脚本运行失败 | 检查 Python 依赖（`pip list | grep pandas`）；确认数据文件存在 | 降级为手工策略判断——人工看 KDJ/MACD/BOLL 日线图 |
| 标的未回测/不在股票池 | 加载 `a-share-comps-analysis` 做可比分析；标注"未经回测验证" | 仅输出框架分析，明确标注风险等级：⚠️ 未经本策略验证 |
| 多指标信号矛盾（如 KDJ超卖但MACD未拐头） | 优先MACD信号（v2.0核心）；标注"信号不一致"等待1-2日确认 | 输出两套分析（看多/看空各一），让用户自己判断 |

### 三级降级：妙想 data() 时间序列 + 自算 BOLL/MACD（push2his + fallback_scan 双挂时）

当 push2his 不可达且 fallback_scan.py 因 dfcf_finance 模块路径问题无法运行时，直接从 Python 调用妙想 `data()` 拉取收盘价时间序列，numpy 自算 BOLL + MACD。**KDJ 因需要高/低价不可算。**

详见 → `references/mx-timeseries-compute.md`

## 操作反例黑名单（不要做什么）

| # | 反模式 | 为什么不要做 | 替代做法 |
|---|---|---|---|
| 1 | **万亿级大盘股用本策略** | 弹性不足，BOLL/KDJ信号钝化 | 排除市值>5000亿标的（工业富联已验证不适合，已清仓+7.6%） |
| 2 | **首日追入买入信号** | 信号常连发2-3日，首日可能是假突破 | 等第2-3日确认后再入场 |
| 3 | **用KDJ覆盖MACD信号** | v2.0核心是MACD趋势确认，KDJ只是辅助 | 信号矛盾时以MACD为准 |
| 4 | **跳过回测直接实盘新标的** | 参数组合对不同股效果差异大（风华+189% vs 利欧+49%） | 新标的必须先跑 `backtest_bb_kdj_macd.py` |
| 5 | **用 v1.0 碰轨卖出** | 踏空主升浪（风华高科碰轨3% vs MACD死叉189%） | 一律使用 v2.0 MACD死叉卖出 |
| 6 | **不刷新数据用旧信号** | 行情变化后MACD金叉/死叉可能已切换 | 实盘前必须通过 dfcf-finance-tool 拉最新日线 |
| 7 | **周五新开仓** | 周末两天黑天鹅窗口，周一可能跳空低开 | 周四/五出现的买入信号等到周一确认再入场；真信号不会只亮一天 |
| 8 | **策略外标的混入主力仓位** | 卓胜微+14%回测=本策略参数不适配；买入逻辑是催化剂赌注不是技术信号 | 策略外操作=娱乐仓规模，认清性质 |
| 9 | **试探仓超过1手** | 未经回测的标的上300股=占用主力仓位资金，机会成本高（卓胜微300股占的仓位如果给佰维/澜起，现在赚的是钱不是教训） | 试探仓铁律：未回测标的初始买1手；确认策略适配后再考虑加仓 |

> 📐 **仓位管理 → 凯利公式：** 详见 `references/investment-framework.md`。回测过的标的用凯利算手数，没回测的凯利没法算——最多1手。
