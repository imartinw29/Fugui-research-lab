# A股可比公司分析（Comparable Company Analysis）

> **核心哲学：先搭对结构，再让数据讲故事。**
> Adapted from Anthropic financial-services comps-analysis, with A-share conventions.

## 适用场景

- 个股估值对标（PE/PB/PS 在同业中的位置）
- IPO / 定增定价参考
- 识别行业内高估/低估标的
- 投资决策委员会材料支撑

**不适用：** 无同行业可比公司的独特标的、严重亏损/ST 公司、业务极度多元化的集团。

---

## ⚠️ 数据源优先级（强制）

1. **第一优先：东方财富 API**（`dfcf-finance-tool` skill）— 行情、财务、估值数据
2. **第二优先：** 公司公告/财报原文（巨潮资讯网）
3. **仅限补充：** Web 搜索 — 绝不作为主要数据源

**数据必须可追溯。** 每个硬编码数字附带来源注释。

---

## 工作流（6 步）

### Step 1：确定可比范围

**使用达尔文进化优化的智能匹配引擎**（从 v2.4 开始）。

不再依赖单一的"同一申万三级行业"粗筛，而是按**业务板块**做多维度匹配：

**匹配逻辑（进化自 Darwinian Evolver，10 组测试用例训练，F1=0.63）：**

1. 拆解目标公司的业务板块（从 `{{description}}` 中提取 2-5 个核心产品/市场板块）
2. 对每个板块，按以下维度匹配可比公司：
   - **核心生产工艺**相同（不只是同一行业分类）
   - **价值驱动因素**相同（成本领先 / 技术溢价 / 品牌 / 渠道）
   - **终端市场/客户画像**重叠
   - **毛利率水平**相似（±10 个百分点内优先）
3. **严格排除：**
   - 同一行业不同子板块的公司（如「石化炼化」≠「聚氨酯制造」）
   - 商业模式不同的公司（分销 ≠ 制造，研发授权 ≠ 生产）
   - 毛利率/体量差异过大（大宗商品 ≠ 特种化学品）
   - 仅在下游/上游有重叠的公司
4. 优先选择：产品线重叠度最高、毛利率相近、营收规模可比的公司
5. **宁缺毋滥：** 如果只有 2-3 家强匹配，就输出 2-3 家，不强行凑 4-8 家
6. 排除 ST / *ST / 停牌超 30 天
7. 排除目标公司自身

**执行方式：** 使用进化后的 prompt 模板调用 LLM，输入目标公司的业务拆解，输出可比公司列表。

### Step 2：获取基础数据

对每个可比公司 + 目标公司，通过 `dfcf-finance-tool` 获取：

**规模与交易数据：**
| 指标 | 来源 | 备注 |
|------|------|------|
| 总市值 | 行情接口 | 亿元 |
| 流通市值 | 行情接口 | 亿元，A 股特色 |
| 股价 | 行情接口 | 元 |
| PE (TTM) | 财务接口 | |
| PB | 财务接口 | |
| 总营收 (TTM) | 财务接口 | 最近四个季度合计 |
| 归母净利润 (TTM) | 财务接口 | |

**经营指标（年度/季度）：**
| 指标 | 公式 |
|------|------|
| 营收增长率 (YoY) | (当期营收 - 去年同期营收) / 去年同期营收 |
| 毛利率 | (营收 - 营业成本) / 营收 |
| 净利率 | 归母净利润 / 营收 |
| ROE | 归母净利润 / 归母权益 |

### Step 3：构建 Excel 分析表

使用 `openpyxl` 创建 `.xlsx` 文件，结构如下：

#### A. 经营指标区

```
Row 1: [行业名称] — 可比公司分析
Row 2: [公司1(代码)] • [公司2(代码)] • ...
Row 3: 数据截至 [日期] | 单位：亿元，除每股指标和比率外
Row 4: (空)
Row 5: 【经营指标】
Row 6: 公司 | 总市值 | 流通市值 | 营收(TTM) | 营收增长% | 毛利率% | 净利率% | ROE%
Row 7+: 数据...
```

**公式规则（不可协商）：**
- 所有派生值（增长率、利润率、ROE）必须是 Excel 公式
- 只有原始输入数据可以硬编码，且**每个硬编码单元格必须有来源注释**
- 示例：毛利率 = `=(C7-D7)/C7` 而非 `0.35`

#### B. 估值倍数区

```
Row N: 【估值倍数】
Row N+1: 公司 | PE(TTM) | PB | PS(TTM) | EV/营收(估)
Row N+2+: 数据...
```

**PS 为 A 股特色补充**（Anthropic 原版以 EV/EBITDA 为主，但 A 股 EBITDA 数据覆盖率低）。

#### C. 统计分布区

在每组指标下方，空一行后添加：

```
最大值:   =MAX(range)
75分位:   =QUARTILE(range,3)
中位数:   =MEDIAN(range)
25分位:   =QUARTILE(range,1)
最小值:   =MIN(range)
```

**需要统计的指标：** 营收增长率、毛利率、净利率、ROE、PE、PB、PS。
**不需要统计的指标：** 绝对值类（市值、营收额）。

#### 视觉规范

- 字体：宋体/微软雅黑，正文 11pt，标题 12pt
- 配色：深蓝标题行 (`1F4E79` 白字) + 浅蓝列标题 (`D9E1F2` 黑字) + 白色数据行 + 浅灰统计行 (`F2F2F2`)
- 无边框，居中对齐，偶数列宽
- 百分比 1 位小数，倍数 1 位小数，金额无小数带千分位
- 目标公司行用**浅金色底** (`FFF8E1`) 高亮

### Step 4：用户确认

每完成一个区块（经营指标 → 估值倍数 → 统计），展示并确认后再进行下一步。

### Step 5：输出与解读

输出 Excel 文件路径 + 关键发现摘要：

- 目标公司在可比组中的位置（高于/低于中位数）
- 估值溢价/折价的可能驱动因素
- 异常值识别与解释
- 局限性声明（样本量、行业周期、数据时点）

### Step 6：交付物清单

- `.xlsx` 可比分析表（含公式和来源注释）
- 分析摘要（Markdown 或飞书消息）

## 与 Anthropic 原版的差异

| 维度 | Anthropic 原版 | A 股版 |
|------|---------------|--------|
| 数据源 | S&P/FactSet/Daloopa MCP | 东方财富 API |
| 估值倍数 | EV/EBITDA 为主 | PE/PB/PS 为主 |
| 行业分类 | GICS | 申万 |
| 市值口径 | 总市值 | 总市值 + 流通市值 |
| 货币单位 | USD | 亿元 |
| EBITDA 覆盖 | 核心指标 | 可选（覆盖率低） |
| 特殊因子 | — | 北向资金、融资余额（可选） |
