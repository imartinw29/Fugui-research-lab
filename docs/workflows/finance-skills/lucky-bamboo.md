## v2.0 策略参数（2026-06-02 验证升级 · v2.9 双参KDJ）

```
布林带: BOLL-M(20), ±2σ
KDJ:    双参接力 — 14,3,3(前哨/灵敏度) + 14,5,5(指挥官/确认度)
MACD:   (12, 26, 9)
周期:   日线

买入信号 = 收盘价 ≤ 布林中轨 AND K<30 AND J<20 AND MACD绿色柱状线开始收窄
卖出信号 = MACD死叉（DIF下穿DEA）  ← v2.0升级：从"碰轨就卖"改为"趋势反转才卖"
```

### KDJ 双参接力（v2.9 · 2026-06-27）

同一观测窗口(14天)配两套平滑参数，分角色使用：

| 角色 | 参数 | 平滑度 | 任务 | 特点 |
|------|------|--------|------|------|
| 🥜 前哨 | 14,3,3 | 低（α=1/3） | 先知先觉，捕捉拐点 | 灵敏度高，拐点快1-2天；假信号多 |
| 🪖 指挥官 | 14,5,5 | 高（α=1/5） | 确认信号，过滤噪音 | 滤掉~30%假信号；确认时可能已涨3-5% |

**实操逻辑：**
1. 14,3,3 发出信号（K<30, J<20）→ 列入观察，不动手
2. 14,5,5 随后确认同步超卖 → 动手
3. 14,3,3 触发但 14,5,5 始终不确认 → 假信号，放弃

两套参数共用同一个 RSV（14天高低价算出的原始值），差别只在 EMA 平滑——温度计同一根水银柱，换了粗细不同的玻璃管。

`quick_scan.py` 四灯全亮后自动显示双参确认状态：
- 🟢 `KDJ 双参确认 — 14,5,5 同步超卖, 非假信号概率高`
- 🟡 `KDJ 前哨触发, 平滑未确认 — 静观其变, 等 14,5,5 跟上或放弃`

**为什么升级：** "碰轨就卖"在趋势行情中会踏空主升浪（风华高科4-6月涨幅200%+，碰轨版只吃到3%）。MACD死叉 = 确认趋势反转才离场，"让利润奔跑"。

回测脚本：`scripts/backtest_bb_kdj_macd.py`（默认使用MACD死叉卖出）
实时筛选：`scripts/screen_buy_signal.py`（拉取东方财富日线，批量判定买入信号）
快速扫描：`scripts/quick_scan.py <secid>` — **v2.9 版**：四灯+双参KDJ(14,3,3前哨+14,5,5指挥官确认)+筹码峰+MACD八形态+股东户数+换手率(市值分档)+PEG+量价背离 · 妙想主力三重降级 · 分拆查询 · 数据窗口 80 天
降级扫描：`scripts/fallback_scan.py <code> [name]` — push2his 不可达时，用 dfcf data() 收盘价计算指标

## 筹码峰（CYQ）作为第五维辅助指标（v2.1 新增 · 2026-06-16）

### 数据来源

通过 DFCF 妙想 API `data()` 查询 `筹码分布 获利盘比例`，返回四个关键指标：

| 指标 | 字段 key | 含义 |
|------|---------|------|
| 平均持仓成本 | `010000_CMPJCB` | 所有持股人的平均买入价（元） |
| 获利盘比例 | `010000_HLP` | 当前价之上有多少持仓在盈利（%） |
| 70%筹码集中度 | `010000_CMFB_461_JZD70` | 含70%筹码的价格区间宽度（%） |
| 90%筹码集中度 | `010000_CMFB_461_JZD90` | 含90%筹码的价格区间宽度（%） |

### 与四灯的配合

```
四灯全亮 + 当前价 < 均价 → 🟢 双重共振（最强信号）
四灯全亮 + 当前价 > 均价 → 🟡 四灯成立但获利盘有兑现压力
四灯不全 + 当前价 < 均价 → 🟡 筹码低位但时机未到
四灯不全 + 当前价 > 均价 → 🔴 都不满足
```

**筹码低位 = 安全边际好**：价格低于平均成本时，套牢盘不愿割肉→卖盘少→反弹阻力小。
**筹码高位 + 获利盘>70%**：获利盘累积→随时可能跑路→即使四灯亮也要警惕。

### 实战案例（格力电器 2026.6）

```
均价 235 / 当前 228 / 获利 25.68% / 70%集中 17.72%
→ 价格在均价之下=筹码低位+安全边际好
→ 四灯全亮+BOLL收口+J=3.36 → 五维共振
```

### 用户看盘对照

- 东方财富 APP 默认显示 BOLL + 筹码峰（CYQ）
- 招商证券默认显示 BOLL-M（无筹码峰）

🔴 **CHECKPOINT · 策略适用性：** 回测结果基于历史数据，不保证未来表现。实盘前确认：当前市场环境是否匹配策略假设（震荡/趋势）？仓位是否在承受范围内？

## 多维信号交叉验证框架（v2.10 · 2026-06-28 · 王奕臻提出）

**核心原则：** 任何单一指标（KDJ/BOLL/MACD/量能/位置）单独发出的信号都是噪音。**至少 3 个维度共振**才能动手，单维度信号必须降级处理。

### 突破验证：量能确认（v2.11 · 2026-06-29 · 澜起实战）

**反直觉：** 前高突破 ≠ 真突破。早盘急拉+同事兴奋+破前高=标准高风险组合。真突破的唯一身份证是**量能**。

**量能验证三步法：**

```bash
# 拉近10天日线量价
python3 -c "
import urllib.request, json
url = 'https://push2his.eastmoney.com/api/qt/stock/kline/get?secid=1.000002&klt=101&lmt=15'
...
# 比较：今日量 / 近N日均量
# >2x → 🚨 异常放量（可能诱多出货）
# 1.2-1.5x → ✅ 正常突破量
# <1x → 🟡 缩量突破（买盘不强，等回踩确认）
"
```

**判断矩阵：**

| 量能 | 价格 | 判断 |
|---|---|---|
| 放量1.2-1.5x | 破前高 | ✅ 真突破概率高 |
| 放量>2x | 破前高 | 🚨 情绪峰值，诱多嫌疑 |
| 缩量 | 破前高 | 🟡 自然换手突破，回踩再加仓 |
| 任意量 | 破前高后回落 | 🔴 假突破确认 |

**关键时点：** 10:30-11:00量价结构决定全天——量能不减+均价线稳住=真突破；量能衰减+大卖单=假突破。

**土办法：** 招商APP日K图看成交量柱高度，跟前天/上前天的柱子比——差不多=正常，高一截=放量，两倍=异常。

**⚠️ 坑：半天量误读为全天量（2026-06-29 实战血泪）**

盘中拉取 push2his 日K数据时，当天K线的成交量是**截至当前时刻的累计量**，不是全天量。直接拿半天量跟全天均量比 → 误判为"缩量" → 方向性错误。

**正确做法：**
1. 先看当前时间——上午/中午拉的数据，只能是半天量
2. 比较基准：半天量 vs 全天均量×50%（半天预估）或全天均量×70%（下午2点预估）
3. **用户实时观察优先于API数据**：用户在招商APP上看到的实时成交量柱子比API数据更新、更直观。API有延迟时，以用户终端数据为准
4. 收盘后再拉日K做全天确认

**实战案例（佰维/蓝思/澜起 2026-06-29）：**
- 上午收盘后拉API → 今天量=均量的53%-94%
- 错误判断：全天量能正常
- 用户纠正："这是半天数据，下午还没开盘"
- 重新计算：半天已到全天均量74% → 全天预估1.5x → **放量确认**

### 六维信号模型

```
买入/卖出决策 = f(位置, 方向, 量能, 节奏, 指标, 时间)
```

| 维度 | 取值 | 信号意义 |
|------|------|----------|
| 位置 | 高/中/低 | 决定方向反转的概率 |
| 方向 | 涨/跌/横 | 当前趋势 |
| 量能 | 放量/缩量 | 买卖双方意愿 |
| 节奏 | 温和涨/暴力涨/急跌/阴跌 | 主力意图 |
| 指标 | KDJ/BOLL/MACD/背驰 | 力学结构 |
| 时间 | 启动/中继/末期 | 所处生命周期 |

### 量能四象限（最易错的维度）

| 状态 | 放量 | 缩量 |
|---|---|---|
| **涨** | 燃料涌入（看位置） | 卖压耗尽（**最佳中继形态**） |
| **跌** | 恐慌出逃（看位置） | 无人接刀（**不要接飞刀**） |

**关键洞察：** **唯一产生新惯性的是放量上涨。** 缩量是等效力互相抵消；下跌不管放量缩量都只代表卖压，不产生新方向。

完整生命周期：
```
放量涨（诞生）→ 缩量涨（延续）→ 放量跌（反转）→ 缩量跌（筑底/停滞）
```

### 节奏三态（主力意图）

| 节奏 | 主力真实意图 | 散户错觉 |
|------|--------------|----------|
| 温和小阳线（每天1-2%） | 悄悄建仓，吃筹码 | "没动静，没机会" |
| 剧烈大阳线/涨停 | 造声势+试盘 | "赶紧追！" |
| 缩量回踩 | 起飞前最后洗盘 | "破位了，跑！" |

**反直觉：** 庄家对敲会产生交易成本（来回约 0.15%），所以庄家一定小量试探而非放量对倒。**"放量温和涨"是真建仓，"缩量剧烈涨"才是诱多嫌疑。**

## 尾盘选股策略（华生·tail-market-scanner）

**核心理念：** 不讲题材板块，只看指标——14:50扫描，符合条件次日开盘卖。胜率略高于抛硬币，真正的优势在**赔率**（尾盘买次日卖，亏损小盈利可能大）。

**两步法：**
1. **海选层** — `clist/get` 全A股：涨幅2-9% + 换手>3% + 量比>1.5 + 去ST
2. **验证层** — `trends2/get` 逐只拉1分钟分时，三维评分（0-100）：
   - 均价线上方时间占比（40分）
   - 下午走势强度（30分）— 斜率+回撤惩罚
   - 尾盘资金托举（30分）— 14:30-14:50稳定性+成交量+支撑

**纪律（王富贵版）：**
- ⏰ 只在14:50跑，不在其他时间
- 🚫 不合格就空仓——不交易也是策略的一部分
- 📉 大盘共振下跌（三四千家跌）→ 空仓日
- 📈 科创板/创业板集体涨停（涨幅前30全>9%）→ 空仓日（温和上涨区间被消灭，不是策略失灵）
- ⚠️ 振幅>8%、疑似炸板 → 自动扣分

**空仓日识别（2026-06-04 实战验证）：** 当日科创板集体爆发，涨幅前30全>11%，20+只封20%涨停板。早上8只候选全部被市场极端化消灭——要么冲板（>9%淘汰）、要么炸板（如永冠新材涨停→收+0.7%）、要么被吸血回落。策略输出0候选是正确保护行为，非故障。

**数据源：** 东方财富公开API（`push2his.eastmoney.com`，HTTPS + urllib，无需密钥）。分钟线/分时数据是妙想Skills API不提供的。详见 → `references/eastmoney-api.md`

**⚠️ 网络坑：** WSL环境下 `requests` 库连 `push2his` 会 `RemoteDisconnected`，`urllib.request` 直连HTTPS可通。脚本已用 `urllib` 规避。`push2.eastmoney.com`（不带his）的HTTP/HTTPS均不可达，必须用 `push2his` 子域。

**脚本：** `scripts/tail_market_scanner.py`
```bash
python scripts/tail_market_scanner.py --top 20 --debug
```

**与主策略（BOLL+KDJ+MACD）的关系：** 尾盘策略是独立的短线工具，与中线回测策略互补——尾盘策略负责"今天买明天卖"的短线机会，中线策略负责"等信号→持仓→MACD死叉离场"的趋势交易。

## 投资框架

**基础设施型垄断**：单价极低但不可或缺，占成品成本<1%，缺了产线停摆。认证周期1-3年，客户不敢换→天然定价权。

**卖耗材 > 卖铲子**：设备卖一次吃十年，耗材用了就得一直买。国瓷卖钛酸钡给村田三星风华——不管谁赢都赚。

详见 → `references/investment-framework.md`

### 适用边界

- ✅ 几百亿~几千亿市值的高弹性标的
- ✅ BB中下轨+KDJ超卖+MACD力竭
- ✅ 有不可替代性的业绩股（认证+工艺+客户惯性）
- ❌ 万亿级大盘股（弹性不足）
- ❌ 纯概念/辟谣股（一字涨跌停跑不掉）
- ❌ 单边下跌趋势（等MACD拐头）

### 策略局限性（回测不代表未来）

回测基于历史数据，以下情况策略可能失灵：

| 风险 | 表现 | 实例 |
|---|---|---|
| **市场风格切换** | 长期单边下跌中，BOLL中轨一路下移，每次抄在中轨都被埋 | 2023-2024年A股阴跌期未覆盖 |
| **MACD延迟确认** | 死叉出现时已回吐大部分利润 | 格力电器第三笔 -30.4%（37→26） |
| **标的市值膨胀** | 同一只股票从几十亿涨到千亿，波动特征改变 | 佰维50→345，低位的信号质量≠高位的信号质量 |
| **行业不匹配** | 参数在存储/MLCC上优化，其他行业未必适用 | 美的集团（射频）回测+14%，远低于存储标的 |

**核心原则：** 策略告诉你概率，不是保证。任何一笔交易都可能亏损——策略的价值在于长期胜率和盈亏比，不在于单笔对错。

### BOLL收口（Squeeze）作为四灯增强信号

当四灯买入信号出现时，检查BOLL带宽是否在收缩——上轨下弯+下轨上翘 = 波动率收敛 = 方向选择在即。

- **收口+四灯+价格贴下轨**：弹簧压紧，突破后往上概率大（下轨在托着价格，空头衰竭+多头蓄力）
- **张口+四灯+价格在下轨附近**：波动还在放大，方向不明，等收口再判断
- **收敛末端**：通常会有一根方向性K线，放量阳线是入场确认，跌破下轨是止损信号

实战示例（格力电器 2026.6）：四灯全亮后BOLL从张口逐步收口（上轨↓下轨↑），配十字星+缩量——底部收敛形态，等放量阳线确认。

### MACD 八形态自动检测（v2.8 · 2026-06-26）

`quick_scan.py` 自动识别八种经典MACD形态，与四灯互补——四灯找买入时机，形态判断趋势结构。

| 分组 | 形态 | 核心逻辑 |
|------|------|----------|
| 零轴下二次金叉 | 小鸭出水 / 海底捞月 | 零轴下金叉→死叉→再金叉，空头衰竭 |
| 零轴上回踩不破 | 空中缆车 / 漫步青云 | 零轴上死叉→DIF不破零轴→再金叉，洗盘结束 |
| 零轴上佛手 | 佛手向上 | 金叉→DIF回踩未死叉→抬头，强势延续 |
| 零轴下天鹅 | 天鹅展翅 | 金叉→DIF回踩未死叉→抬头，弱势转强 |
| 穿零轴粘合 | 空中缆绳 | 零轴下金叉→上穿→粘合→分离抬头 |
| 零轴下粘合 | 海底电缆 | 零轴下长期粘合→金叉，底部蓄势 |

实战：格力电器 2026.6.26 检测到「空中缆车(零轴上回踩不破)」——趋势结构健康，但四灯 0/4 无买点，信号互补。

详见 → `references/macd-8-patterns.md`

### 半日数据陷阱（2026-06-29 实战教训）

拉取东方财富 API 日K线数据时，**当天下午15:00前返回的是半日数据，不是全天数据**。将半日量与全天均量比较会得出完全相反的结论：

- ❌ 午盘拉数据，看到"今量 < 均量" → 误判为缩量
- ✅ 午盘拉数据，今量已达均量74% → 全天预估1.5x放量

**规则**：午盘/盘中拉日K数据时，必须标注"半天数据"，且全天预估量 = 今量 × 2（粗略）。下午15:00后拉才是全天数据。

### 数据源优先级（强制 · 2026-06-29 实战校准）

**妙想 dfcf data() 五合一套餐优先于 push2his。** 日级别及以上行情数据一律先走妙想：

```
1. 妙想 data() OHLC五合一 → 首选（326269/325898/326339/326386/326699）
2. push2his kline/get → 仅当妙想不支持时（分钟线/分时/次新股）
```

**为什么**：妙想返回干净实时、不抽风；push2his 间歇性全站宕机（本日多次 RemoteDisconnected）。

**验证**：每天第一次拉数据前，先 `curl` 测 push2his 存活；存活则可用，宕机则直接降级妙想（不重试 push2his 三次）。

### 数据源故障时的实用策略

push2his 间歇性不可达 + dfcf data() 对A+H股返回混合价格（如澜起同时返回A股228和H股345两个价格序列）。当两个数据源都不可靠时：

1. **优先信任用户终端数据**——用户在看盘软件上的实时数据比API更准。用户屏幕上的价格、成交量、KDJ、布林数值 = 第一手事实
2. **API数据存在延迟**——push2his日K的当天数据是累计量（非全天），盘中容易误读；收盘后才能拿到全天完整数据
3. **不要反复重试失败的数据源**——两次失败后直接用用户提供的数字计算
4. **web_search 可查收盘价**（雪球/新浪/Yahoo），但日级别K线细粒度数据仍以用户为准

**数据信任层级（从高到低）：**
1. 用户终端实时截图/口述 → 最高优先级，无需质疑
2. push2his API（收盘后完整日K）→ 二级
3. push2his API（盘中累计量）→ 三级，需标注"半天数据"
4. web_search/雪球/新浪 → 四级，仅验证用
5. 模型训练数据残余 → **禁止使用**，如"京东方A约30元"（实为102元，2026-06-29事故）

## 异常与降级策略

数据来源优先级：**东方财富公开 API（`push2his`，quick_scan.py 首选）→ 妙想 API（`dfcf_finance.py data()`，降级方案）**。公开 API 返回完整 OHLC，可一次算完四灯；妙想 API 每条 query 只返回一个指标，需要三次独立调用。

### push2his 不可达时的妙想降级（重要）

当 quick_scan.py 报 `Remote end closed connection without response` 时，按以下步骤降级：

1. **单独查询收盘价**：`tool.data('000002 澜起 收盘价 2026-04-01至2026-06-10')`
2. **单独查询最高价**：同上但 query 为「最高价」
3. **单独查询最低价**：同上但 query 为「最低价」

每个查询返回的 table 结构中，实体 key 是**数字 ID**（如 `325898`），不是股票名称。headName 存日期（格式 `2026-06-09(日)`），值带「元」后缀。需要 `re.sub(r'[元港,日\(\)]', '', s)` 清洗后转 float。

**关键坑：** 一次 query 同时请求「收盘价 最高价 最低价」时，返回 2 个 table——table 0 是汇总统计（区间高低点 + 日期），table 1 只有收盘价序列。高点/低点不会出现在同一 query 中，必须分三次调用。妙想 data() **不支持 OHLC 四合一查询**。

数据反转：返回的序列是**最新在前**（降序），需要 `[::-1]` 反转为升序后才能计算指标。

### 妙想 data() 返回错误价格（A/H 股混淆）

当股票有 A+H 双重上市时（如格力电器 000002 + 06809），`data()` 的 table 中可能同时包含 A 股和 H 股数据。H 股价格通常远高于 A 股（溢价），若误取 H 股价格，会导致 BOLL/MACD/KDJ 全部失真。

**防御措施：** 解析 table 时优先匹配含 `.SZ` /「A股」/股票代码（不含港）后缀的 key；若遇到 H 股价格（通常以「港」结尾），跳过。quick_scan.py 直接从 push2his 拉取，不存在此问题。**fallback_scan.py 已修复（2026-06-12）：** ① 跳过含「港元」的 table（妙想 data() 对 A+H 股票返回两个 table，Table 0=HKD、Table 1=CNY）；② `parse_prices()` 过滤含「港」的值；③ 补了缺失的 `import os`。

### 网络连通性

从 WSL/Linux 环境访问东方财富 API 时，`requests` 库可能连接失败（`RemoteDisconnected`），但 **`urllib.request.urlopen` 同 URL 可通**。所有调用东财公开接口的脚本（quick_scan、backtest、tail_market_scanner）必须使用 `urllib` 而非 `requests`。

**已验证可通的子域名：** `push2his.eastmoney.com`（HTTPS）。`push2.eastmoney.com`（HTTP）不可达。该域名**间歇性全站宕机**——服务端不可用，与客户端 SSL/网络/Tailscale 均无关。

**🔬 诊断 push2his 是否存活（2026-06-12 验证）：**

```bash
# 快速存活检测（curl 通=服务在线，curl 不通=服务端宕机）
curl -s --max-time 5 "https://push2his.eastmoney.com/api/qt/stock/kline/get?secid=1.000002&fields1=f1&fields2=f51&klt=101&fqt=0&end=$(date +%Y%m%d)&lmt=1"
```

**🔬 诊断流程（2026-06-12 验证）：**

```bash
# Step 1: 存活检测
curl -s --max-time 5 "https://push2his.eastmoney.com/api/qt/stock/kline/get?secid=1.000002&fields1=f1&fields2=f51&klt=101&fqt=0&end=$(date +%Y%m%d)&lmt=1"
```
- curl 返回 JSON → 服务在线，继续 Step 2
- curl 返回空/超时 → 服务端宕机，直接走妙想降级

**Step 2: 如果 curl 能通但 quick_scan.py 仍然报 error 102：** 运行 `scan()` 函数直接调用（绕过命令行入口的间歇问题）：
```python
python3 -c "exec(open('scripts/quick_scan.py').read().replace('if __name__', 'if False and __name__')); scan('1.000002')"
```

**Step 3: Python urllib 的 TLS 问题（WSL特有）：** `quick_scan.py` 已内置 `ssl._create_unverified_context()`（2026-06-12 修复）。症状：服务在线但 Python 报 `RemoteDisconnected`，curl 正常。根因：Python 默认 TLS 握手与东方财富 IIS 服务器不兼容。

若子技能或数据源不可用，按以下三段式 fallback：
1. `quick_scan.py`（push2his，含高/低/收价 → 完整 KDJ+BOLL+MACD）→ 首选
2. `quick_scan.py` 报 error 102 但 curl 能通 → exec() 直调 `scan()` 函数（绕过命令行入口间歇问题）
3. `fallback_scan.py`（妙想 API 收盘价 → numpy 算 BOLL/MACD，高/低价用 close±2% 估算 KDJ）→ push2his 不可达时
4. 裸调妙想 API 分别查询收盘价/最高价/最低价（三次 API 调用，完整 KDJ）→ fallback_scan 也失败时
4. 妙想 API 返回的时间序列数据提取模式（重要）：
   - `data()` 用中文"至"连接日期：`'000001 收盘价 2026-04-01至2026-06-10'`
   - 返回 `table` 字典，**entity key 是数字 ID（如 '325898'），不是股票代码或名称**
   - `headName` 存日期（格式 `'2026-06-09(日)'`），需 `replace('(日)','')` 清洗
   - 价格数组含 `'元'` 后缀，需 `replace('元','')` 后转 float
   - 日期顺序为最新在前，需 `[::-1]` 反转为时间顺序
   - 收盘价/最高价/最低价需分开三次查询，单次混合查询不返回多表注意：push2his 可能间歇性全站不可达（RemoteDisconnected），此时需走三级降级。

若子技能或数据源不可用，按以下三段式 fallback：

| 触发条件 | 一线修复 | 仍失败兜底 |
|---|---|---|
| dfcf API 无响应/返回空 | 重试1次，间隔3秒；切换备用数据源（web_search查行情） | 告知用户"数据不可用"，使用上次缓存数据（标注日期） |
| push2his RemoteDisconnected | ① curl 快速存活检测；② 在线 → `quick_scan.py` 内置 SSL fix 已处理；③ 宕机 → `fallback_scan.py <code> <name>`（妙想收盘价+估算KDJ） | 裸调妙想 API 分三次查收/高/低价完整算KDJ |
| quick_scan.py error 102（但 curl 能通） | exec() 直调 `scan()` 函数 | 同 push2his 降级 |
| 回测脚本运行失败 | 检查 Python 依赖（`pip list \| grep pandas`）；确认数据文件存在 | 降级为手工策略判断——人工看 KDJ/MACD/BOLL 日线图 |
| 回测脚本运行失败 | 检查 Python 依赖（`pip list | grep pandas`）；确认数据文件存在 | 降级为手工策略判断——人工看 KDJ/MACD/BOLL 日线图 |
| 标的未回测/不在股票池 | 加载 `a-share-comps-analysis` 做可比分析；标注"未经回测验证" | 仅输出框架分析，明确标注风险等级：⚠️ 未经本策略验证 |
| 多指标信号矛盾（如 KDJ超卖但MACD未拐头） | 优先MACD信号（v2.0核心）；标注"信号不一致"等待1-2日确认 | 输出两套分析（看多/看空各一），让用户自己判断 |
