# 为什么 Skill 比 Prompt 更值钱

Prompt 很重要，但很多时候，它只解决一次。

Skill 不一样。Skill 解决的是：

- 什么时候用
- 先做什么
- 后做什么
- 卡住了怎么办
- 做完以后如何复用

也就是说，prompt 更像一句聪明的话，skill 更像一套能落地的工作方法。

## Prompt 的上限在哪里

一个 prompt 可以写得很漂亮，也能短时间拉高输出质量。

但它常见的问题是：

- 只在一个场景里有效
- 对输入要求隐含很多前提
- 无法自然吸收失败经验
- 很难长期维护

你下一次再遇到相似问题，往往还得重新调。

## Skill 的价值是什么

Skill 更像程序化记忆。

一个好的 skill 通常会包含：

- 触发条件
- 输入要求
- 执行步骤
- 常见坑
- 验证方式
- 模板或参考文件

这意味着它不仅告诉你“说什么”，还告诉你“怎么做”。

## 为什么我更想沉淀 Skill

因为真正值钱的，不是某一次生成得有多漂亮，而是：

- 这个方法下次还能不能复用
- 它能不能在不同对象上迁移
- 它有没有因为失败被修正过

换句话说，skill 更接近资产，prompt 更接近材料。

## 一个简单判断标准

如果某个 prompt：

- 已经重复用过几次
- 每次都要补相似说明
- 一旦失败就得手动解释很多上下文

那它大概率已经值得升级成 skill。

## 一句话总结

> Prompt 让一次输出更聪明，Skill 让整个系统越来越稳。
