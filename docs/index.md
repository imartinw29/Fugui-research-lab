# Fugui Research Lab

> 一个持续沉淀方法论、研究框架、智能体工作流与实践笔记的个人研究实验室。

有些内容只适合刷过去，有些内容适合存下来反复用。

这个站更关心后者。

我想在这里长期记录的，不是热点，不是空泛的观点，也不是用篇幅假装深度的长文，而是那些真正能留下来的东西：

- 能帮助理解复杂问题的方法
- 能复用到下一次任务里的工作流
- 能从 prompt 升级为 skill 的结构化经验
- 值得反复修订、逐步变稳的判断与笔记

很多时候，真正稀缺的不是信息，而是把信息组织成行动与判断的能力。这个实验室的存在，就是为了给这种能力留一个公开、持续、可迭代的容器。

## 你会在这里看到什么

### Methods
研究方法与分析框架。

这里会放那些可以反复使用的方法，比如：

- 横纵分析法
- 问题拆解与变量识别
- 深度研究的结构搭建方式

它们的共同点是：不是只说“应该怎么想”，而是尽量说清“具体怎么做”。

### Workflows
可执行的工作流、SOP 与智能体协作方式。

我对 workflow 的兴趣，不在于把流程写得很满，而在于让它在真实场景下更稳。一个值得留下来的工作流，应该能解释：

- 什么时候用
- 为什么这样排顺序
- 卡住以后怎么退路
- 做完以后如何沉淀

### Notes
值得回看的短文、判断与工作笔记。

这些内容不一定都很长，但至少应该有一个前提：以后再看时，仍然有用。

## 为什么要做这个实验室

因为我越来越不相信“多收集一点”天然会通向“理解更深一点”。

真正起作用的，往往是另外几件事：

- 把问题定义清楚
- 把结构搭起来
- 把失败经验留下来
- 把一次性的做法改造成可复用的方法

如果一个地方能持续帮助我做这几件事，它就值得被维护。

## 这个站的工作方式

这里的内容会尽量遵循几个标准：

1. **可复用**：最好能变成模板、清单、skill 或 SOP。
2. **可追溯**：关键判断尽量给出事实基础、时间节点或上下文。
3. **可迭代**：允许先有一个版本，再慢慢修成更稳的版本。
4. **有观点**：不靠模糊话术维持体面。

## 一句话总结

> 这里不想生产更多内容，只想沉淀更少但更耐用的方法。
