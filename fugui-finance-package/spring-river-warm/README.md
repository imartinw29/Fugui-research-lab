# 春江水暖 — A股个股深度估值引擎

> 春江水暖鸭先知。估值不是算一个数字，是理解市场在押注什么、什么条件下这个赌注会变。

## 架构哲学

**脚本能做的不用 skill，skill 能做的不用 LLM。**

| 层 | 工具 | 做什么 |
|---|------|--------|
| 🔧 脚本 | `valuation_rules.py` | 利润质量/成长质量/估值驱动检测 + Regime 自动分类 |
| 🔧 脚本 | `clean_financials.py` | 东方财富混杂格式 → float，自动计算 QoQ/YoY/TTM |
| 📋 框架 | SKILL.md（本文件） | 七章报告结构 + 投资语言模板 |
| 🤖 LLM | 最后一步 | 把前面算出来的数据写成人话 |

## 管线

```
数据采集 → 数据清洗 → 推理规则 → Regime分类 → 多方法估值 → 预期差 → 投资语言
   ↑            ↑           ↑
dfcf_finance  clean_     valuation_
   .py      financials  rules.py
```

## 核心脚本

### `valuation_rules.py` — 推理规则引擎

| 规则 | 触发条件 | 结论 |
|------|---------|------|
| 利润质量 | CFO/净利润 < 0.8 | 利润质量偏弱 |
| 应收异常 | 应收增速 > 营收增速 +20pct | 渠道压货风险 |
| 成长质量 | 营收+40% 且毛利率-5pct | 依赖低价扩张 |
| 存货积压 | 存货增速 >> 营收增速 | 减值风险 |
| 估值驱动 | PE↑ + EPS↓ → 纯估值扩张 | PE↑ + EPS↑ → 业绩驱动 |

**Regime 分类：**

```python
from valuation_rules import RuleEngine

regime = RuleEngine.classify_regime(
    gm_volatility=0.42,   # 近3年毛利率极差
    revenue_growth=0.80,  # 近3年营收CAGR
    roe=0.15,
)
```

| Regime | 主估值方法 | 避免 |
|--------|----------|------|
| 周期股 | PB + 库存周期位置 | PE（顶部利润最大=PE最低，具误导性） |
| 高景气成长 | PS + Forward PE | DCF（假设过于敏感） |
| 稳定白马 | DCF + ROIC | PS（成熟期不适用） |

### `clean_financials.py` — 数据清洗

- `parse_dfcf_value()`：东方财富混杂格式("68.14亿元"/"42.22%"/"-5.748元") → float
- `compute_derived()`：自动计算 QoQ / YoY / TTM / 毛利率变化 / 存货周转

## 报告结构（七章）

```
1. 公司概况与业务结构
2. 核心业务表现与增长逻辑
3. 最新财报数据解读
   ├ 核心财务指标表（YoY/QoQ）
   ├ 盈利能力 + 运营能力 + 偿债能力
   ├ 资本回报与现金流（ROE拆解/ROIC/FCF剪刀差）
   ├ 推理规则检查结果（来自 valuation_rules.py）
   └ 研发投入
4. 估值（核心）
   4.1 隐含PE校验与市场定价拆解
       ├ 动态PE vs TTM PE（必须同时报告，解释差异）
       └ 历史PE Band
   4.2 可比公司估值锚定（引用 peer-comps-builder 输出）
       └ 筹码结构（北向/融资/机构）+ 解禁压力
   4.3 Regime 分类 + 方法选择（来自 valuation_rules.py）
   4.4 前瞻估值法（乐观/中性/悲观 三情景）
       ├ 远期PE法
       ├ PS辅助法
       └ 分部估值法（触发条件：毛利率差>15pct 或 增速差>20pct）
   4.5 估值自洽性校验
       ├ 多方法交叉
       ├ DCF隐含增速反推（FCF为正才启用）
       └ 估值驱动来源（来自 valuation_rules.py）
   4.6 预期差分析
   4.7 市场在押注什么
5. 风险提示
6. 估值结论
   ├ 估值可信度评分（高/中/低）
   ├ 核心估值区间
   ├ 催化剂时间线
   └ 观测指标与阈值（持有/减仓/离场）
7. 免责声明
```

## 估值可信度评分

```
✅ 高确定：行业稳定 + 连续盈利 + 现金流健康 + 无单一客户依赖
⚠️ 中等：具备3/4
❌ 低确定：强周期/现金流为负/政策敏感/技术路线存疑
```

评分本身就是估值结论的一部分。低确定标的的宽估值区间是信号，不是模型缺陷。

## ⚠️ 动态PE vs TTM PE 陷阱

东方财富 `f162` 的"动态PE" = 市值 / (最新单季净利 × 4)。**周期股利润暴增/暴跌时严重失真：**

```python
# 平安银行示例
ni = [28.99, 8.53, 0.3041, -2.258, -1.975]  # Q1→Q1上年
ttm_ni = ni[0] + ni[1] - ni[4]  # = 39.5亿
ttm_pe = mcap / ttm_ni            # = 36.7x

# 动态PE = 市值/(ni[0]×4) = 12.5x，差了3倍！
```

**报告中必须同时列出动态PE和TTM PE，解释差异原因。**
